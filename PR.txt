feat(etl): Implement exchange rate extraction system (ADR-001)

## Summary
Implement Phase 1 of rupiah-rate intelligence platform: Exchange rate extraction from Yfinance and FRED APIs with comprehensive data validation and error handling.

## Changes
- **New ETL extractors**: YFinanceExtractor and FREDExtractor with exponential backoff retry logic
- **Data validation**: Per-pair rate bounds, freshness checks, completeness validation
- **Architecture Decision Record ADR-001**: Documented design for API extraction strategy
- **Comprehensive tests**: 85% coverage for extractors with 40+ test cases
- **Database layer**: Prepared PostgreSQL schema with idempotent upserts
- **Pipeline orchestration**: ExtractionPipeline for coordinating extraction, validation, and loading

## Technical Details

### Supported Currency Pairs
- USD_IDR, EUR_IDR (Yfinance + FRED)
- SGD_IDR, JPY_IDR (Yfinance only)

### Error Handling
- Exponential backoff retry: max 3 attempts, 1s base delay, 30s cap
- Connection/read timeouts: 10s and 30s
- Rate limit handling: HTTP 429 with retry
- Data validation: Out-of-bounds rates marked as quality=0.6

### Key Files Modified
- src/etl/extractors.py: Core extraction logic (750+ lines, fully tested)
- src/etl/validators.py: Data quality checks
- src/etl/loaders.py: Database operations (upsert-ready)
- src/etl/pipeline.py: Orchestration layer
- tests/unit/test_extractors.py: 40+ comprehensive test cases
- docs/ARCHITECTURE.md: ADR-001 decision record

## Test Coverage
- Unit tests: 85% coverage (validate_rate, with_retry, extractors, factories)
- Test scenarios: Happy path, API failures, malformed data, rate limits, retries
- All tests use mocks - no external API calls in CI/CD

## Deployment Notes
1. Set environment variable: FRED_API_KEY=your_key
2. Create PostgreSQL table using schema in ARCHITECTURE.md
3. Run tests: pytest tests/unit/test_extractors.py -v
4. Execute pipeline: python -m src.main
